# 📅 일정 조율 및 통합 관리 시스템 (Scheduler Release)

정기 오케스트라 합주 조율, 클래스 세션 및 다양한 일정 예약을 체계적으로 관리할 수 있는 **Full-Stack (Go + React)** 일정 조율 시스템입니다. 

본 프로젝트는 고도의 디자인 품질, 미학적 색상 조합, 그리고 직관적인 타임라인 그리드를 갖추었으며, Firebase 연동과 Docker 기반 분산 배포 환경을 지원합니다.

---

## 🏗️ 전체 아키텍처 개요

본 애플리케이션은 **프론트엔드(React/TypeScript)**와 **백엔드(Go/Gin)**가 분리된 고성능 아키텍처를 지향합니다.
* **로컬/샌드박스 개발 환경**: 하나의 풀스택 흐름을 위해 `server.ts` (Express)가 정적 파일 서빙과 실시간 API 모킹/프록시를 담당하여 Port 3000에서 완벽하게 통합 구동됩니다.
* **릴리즈/배포 환경**: 멀티 스테이지 빌드가 적용된 **Docker**를 통해 독립적으로 컨테이너화되며, 프론트엔드의 **Nginx Reverse Proxy** 설정을 거쳐 안전하게 백엔드 컨테이너(`backend-service:3000`)로 API 요청이 라우팅됩니다.

---

## 📂 파일별 목적 및 역할 상세 설명

### 1. 루트 영역 (Root)

| 파일 / 폴더 | 목적 및 역할 설명 |
| :--- | :--- |
| **`server.ts`** | AI Studio 샌드박스 환경(Port 3000)에서 React 빌드 정적 파일을 서빙하고 실시간 API 요청을 통합 처리하는 초경량 Express 서버입니다. Go 백엔드 데이터 구조와 1:1 대응되는 메모리 DB 세션을 내장하여 샌드박스 내에서 완벽한 동작을 보장합니다. |
| **`package.json`** | 프로젝트 루트의 의존성(Express, Cors, TypeScript) 및 실행 스크립트(`npm run dev`, `npm run build`, `npm run start`)를 정의합니다. |
| **`docker-compose.yml`** | 운영(Release) 환경에서 백엔드 고랭 서비스(`scheduling_backend`)와 프론트엔드 Nginx 서비스(`scheduling_frontend`)를 멀티 컨테이너 네트워크로 묶어 간편하게 일괄 실행하는 오케스트레이션 명세서입니다. |
| **`metadata.json`** | AI Studio 플랫폼의 앱 명칭, 설명, 기기 권한 및 프레임 허용 제어 정보를 정의하는 메타데이터 파일입니다. |

---

### 2. 백엔드 영역 (`/backend/`)
Golang 1.22+ 기반으로 구현되어 초고속 응답 속도와 뛰어난 안정성을 제공합니다.

| 파일명 | 목적 및 역할 설명 |
| :--- | :--- |
| **`main.go`** | Go 애플리케이션의 엔트리 포인트입니다. Gin 웹 프레임워크 엔진을 초기화하고, CORS 정책 설정, Firestore DB 인스턴스 주입 및 백엔드 라우터(Router Group)를 정의하여 3000 포트에서 수신 대기합니다. |
| **`auth.go`** | Firebase Authentication 자격 증명 토큰(`Bearer ID Token`)을 검증하는 미들웨어가 정의되어 있습니다. 또한 개발/샌드박스 환경에서 원활한 테스트를 위해 `dev-token-` 접두사를 이용한 **자격 증명 우회(Bypass) 메커니즘**을 탑재하고 있습니다. |
| **`db.go`** | Google Cloud Firestore 클라이언트 라이브러리를 사용해 NoSQL 데이터베이스 연결 상태를 이니셜라이징하고 관리합니다. |
| **`models.go`** | 백엔드 데이터 파이프라인에서 사용하는 구조체(Struct) 정의 파일입니다. 일정 세션(`Session`), 설문 제출 게스트(`Guest`), 카테고리 태그(`Category`), 감사 로그(`AuditLog`) 등의 구조와 JSON 태그 매핑이 명시되어 있습니다. |
| **`handlers.go`** | 비즈니스 로직의 핵심 구체 클래스입니다. 세션 생성/수정/삭제(물리 삭제 및 보관함 이동용 소프트 삭제), 카테고리 태그 관리 및 게스트가 제출한 시간대 설문 조사를 수집/병합하는 컨트롤러 핸들러 함수들이 모여 있습니다. |
| **`Dockerfile`** | **Multi-Stage Build** 패턴을 따르는 백엔드 이미지 제작 파일입니다. 1단계(`golang:alpine`)에서 고성능 정적 바이너리로 컴파일한 후, 2단계(`alpine:latest`) 초경량 프로덕션 이미지로 복사하여 컨테이너 크기를 최소화하고 보안성을 높였습니다. |

---

### 3. 프론트엔드 영역 (`/frontend/`)
React, TypeScript 및 Tailwind CSS 조합으로 최신 트렌드의 고대비 다크 테마(Space Theme) 인터페이스를 제공합니다.

| 파일 / 폴더 | 목적 및 역할 설명 |
| :--- | :--- |
| **`src/App.tsx`** | 프론트엔드의 최상위 메인 뷰 컴포넌트입니다. 관리자 대시보드, 상세 현황 분석 패널, 게스트 설문 참여 폼 등의 화면 전환 상태(State)를 총괄 제어하며 백엔드 API와의 비동기 통신(`fetch`)을 통합 조율합니다. |
| **`src/components/`** | 대시보드 리스트(`Dashboard.tsx`), 상세 정보 및 합주/회의 최종 시간대 분석기(`Detail.tsx`), 게스트 시간 다중 선택 매트릭스(`Survey.tsx`), 팝업창 관리(`Modals.tsx`), 이메일 간편 패스워드리스 로그인(`Login.tsx`), 그리고 상호작용 알림(`Toast.tsx`) 등 재사용이 가능하고 잘 설계된 인터페이스 모듈들이 분할 정의되어 있습니다. |
| **`src/utils/`** | 캘린더 주차별 일자 계산 연산 유틸리티(`date.ts`)와 일정 확정 결과를 외부 파일로 추출할 수 있도록 돕는 다운로드 처리 코드(`export.ts`)가 배치되어 있습니다. |
| **`nginx.conf`** | 프론트엔드 배포를 위한 핵심 설정 파일입니다. 단일 페이지 애플리케이션(SPA)의 HTML5 History 라우팅 문제를 극복하기 위해 `try_files` 처리를 수행하고, 브라우저가 전송하는 `/api/` 요청을 백엔드 서버 주소(`http://backend-service:3000`)로 매끄럽게 프록싱(Reverse Proxy)합니다. |
| **`Dockerfile`** | Node.js 컴파일 스테이지를 거쳐 산출된 고성능 번들 파일(`dist/`)을 실무용 Nginx 웹 서버의 루트 서빙 경로로 복사하여 프로덕션 이미지로 구동시키는 프론트엔드 배포 자동화 파일입니다. |

## 🚀 배포 및 로컬 실행 가이드

### 1. 개발 샌드박스 (통합 포트 3000)
본 샌드박스 환경에서는 아래 명령을 통해 간편하게 통합 구동을 테스트할 수 있습니다.
```bash
# 전체 의존성 설치 및 프로덕션 빌드 연동
npm install
npm run build

# 통합 Express + React 환경 시작
npm run dev
```

### 2. Docker Compose 활용 배포 (Release 환경)
백엔드 고랭 애플리케이션과 프론트엔드 Nginx를 분리하여 실제 운영 서버에 기동하려면 루트 디렉토리에서 아래 명령을 실행합니다.
```bash
# 전체 컨테이너 무중단 빌드 및 백그라운드 구동
docker-compose up --build -d
```
이후 사용자는 `http://localhost:3000`을 통해 접속하여 완벽히 통합된 프로덕션 릴리즈 앱을 만나볼 수 있습니다.
